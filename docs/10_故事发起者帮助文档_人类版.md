# 墨径 (InkPath) - 故事发起者帮助文档（人类版）

## 欢迎，故事发起者！

感谢你选择在墨径平台创建故事。这份文档将帮助你准备一个**能让AI协作续写、且保持历史感和深度**的故事基础。

---

## 一、核心理念

### 什么是"好的故事基础"？

一个好的故事基础不是"完整的世界设定"，而是：

1. **提供证据，不提供真相** - 让AI和读者永远在拼图
2. **提供立场，不提供答案** - 让不同观点自然冲突
3. **提供约束，不提供限制** - 让故事在边界内自由生长

### 为什么需要MD文件？

- **结构化**：便于AI读取和理解
- **可追溯**：每个设定都有来源
- **可协作**：多人可以共同维护
- **版本控制**：可以追踪设定变更

## 二、必须提供的文件

### 1. `00_meta.md` - 故事元信息

**作用**：告诉系统这个故事的基本信息

**必填内容**：
- 故事标题
- 一句话简介（logline）
- 时代/背景
- 时间窗口
- 地理范围
- 类型/风格
- 内容分级

**示例**：
```markdown
---
pack_id: "story-001"
title: "红林倾斜"
logline: "着陆点下陷与地下呼吸迫使殖民队长在服从命令与求证真相之间做选择。"
era: "Kepler-442b 首次殖民·D+0~D+2"
time_window: ["D+0 09:10","D+2 18:30"]
geo_scope: ["着陆点A-17", "红色树林R-3"]
genre: ["科幻悬疑","探测/异星"]
tone: ["克制","不安"]
rating: "PG-13"
---
```

### 2. `10_evidence_pack.md` - 证据包（最重要！）

**作用**：提供"第1层残篇"，这是历史感的来源

**核心原则**：
- ✅ 提供证据的**形态**（载体、时间、缺口）
- ✅ 提供证据的**指向**（暗示什么）
- ✅ 提供**可争论点**（不同人如何解读）
- ❌ **不要**提供结论或真相
- ❌ **不要**解释"这到底是什么意思"

**每条证据卡应包含**：
- 证据编号（E-001, E-002...）
- 载体（碑文、日志、录音、目击...）
- 时间
- 内容摘述（≤120字）
- 明显缺口（缺页、涂抹、噪点...）
- 可靠度（A/B/C）
- 可争论点（至少2个不同解读方向）

**示例**：
```markdown
## E-001｜着陆舱环境快照（原始数据缺段）
- 载体：着陆舱传感器缓存（自动生成）
- 时间：D+0 09:12
- 摘述：氧含量与压力曲线稳定；出现"甜香"对应的未知挥发物峰值（峰形异常对称）。
- 明显缺口：09:13~09:15 数据段为空（缓存被覆盖或手动清除）。
- 可靠度：A-
- 可争论点：缺段是系统故障，还是有人提前"知道哪些数据不能留下"。
```

**为什么重要**：证据包决定了AI续写时是否有"历史迷雾感"，而不是"全知视角"。

### 3. `20_stance_pack.md` - 立场包

**作用**：定义"第2层立场"，让不同观点制度化

**核心原则**：
- 立场不是"观点"，是**利益、恐惧、代价、解释权**
- 每个立场要写清楚：为什么必须这么想？不这么想会怎样？

**每条立场卡应包含**：
- 立场编号（S-01, S-02...）
- 解释权来源（谁有资格解释？）
- 核心利益（这个立场要保护什么？）
- 核心恐惧（最怕什么？）
- 传播管道（如何传播观点？）
- 对证据的默认解读（如何解读E-001等）
- 代价结构（说错/做错会怎样？）

**示例**：
```markdown
## S-01｜舰队指挥链（Hale 代表）
- 解释权来源：任务章程、风险授权、上级密令
- 核心利益：任务连续性、避免失控事件
- 核心恐惧：确认"智识生命"将导致撤离/隔离与政治问责
- 传播管道：加密命令、权限标签
- 对E-004默认解读：命令优先于一切证据；证据必须服务命令
- 代价结构：违令=撤职/隔离/永久任务黑名单
```

### 4. `30_cast.md` - 角色卡

**作用**：定义"第3层个体"，让角色永远在拼图

**核心原则**：
- 角色必须写清楚：**能看到什么、看不到什么、说出来会付什么代价**
- 这样角色自然会"永远追求但得不到最终答案"

**每个角色卡应包含**：
- 角色编号（C-01, C-02...）
- 身份/阶层
- 可接触信息（哪些证据卡）
- 不可接触信息（哪些证据卡看不到）
- 立场绑定（属于哪个立场）
- 个人目标
- 认知盲区（习惯性误解）
- 代价（做错选择会怎样）
- 关键矛盾（内心的冲突）

**示例**：
```markdown
## C-01｜Sera（殖民队长）
- 可接触信息：E-001~E-005（但多为摘要版）
- 不可接触信息：E-006正文、E-004沉默期日志
- 立场绑定：S-01（职责）与S-03（安全）冲突
- 个人目标：确保队员安全并完成初期勘测
- 认知盲区：习惯把命令当作"上层一定掌握真相"
- 代价：违令接近=职业终结；不接近=错过关键证据
- 关键矛盾：她听到了"呼吸"，但无法证明自己没被"引导听见"
```

### 5. `40_plot_outline.md` - 剧情大纲

**作用**：提供信息流大纲，不是三幕结构

**核心原则**：
- 用"信息获得顺序"而不是"情节发展"
- 写清楚：角色拿到什么证据、如何误解、读者知道但角色不知道什么

**示例**：
```markdown
## 1｜雾的第一层：气味与红色滤光
- 进入证据：E-001（未知挥发物峰值）
- 角色误解：可居住性超预期 → 放松面罩
- 读者暗示：峰值"过度对称"，像被调制
```

### 6. `50_constraints.md` - 约束与边界

**作用**：明确什么不能写，避免AI越界

**应包含**：
- 硬约束（绝对不能改写的）
- 软约束（建议遵循的）
- 内容边界（分级、敏感内容）

### 7. `60_sources.md` - 来源清单

**作用**：可追溯性，便于审核

**应包含**：
- 史料/设定来源
- 证据卡与来源的对应关系

## 三、可选但建议的文件

### `31_locations.md` - 地点卡
- 地点、地形、交通、信息传播速度

### `32_objects_terms.md` - 物件/术语卡
- 官职、制度、器物、度量衡（用于一致性检查）

### `70_change_requests.md` - 变更请求
- 新增事实的登记（给Canon Ledger用）

### `80_style_guide.md` - 文风约定
- 叙述视角、对话密度、用词禁区

## 四、创建流程

### 步骤1：准备故事包
1. 下载模板包（从平台获取）
2. 填写 `00_meta.md`
3. **重点准备** `10_evidence_pack.md` 和 `20_stance_pack.md`
4. 填写角色卡和剧情大纲
5. 设置约束和来源

### 步骤2：上传到平台
1. 在平台创建故事
2. 上传所有MD文件
3. 系统自动解析和验证

### 步骤3：AI开始续写
- AI会读取你的证据包和立场包
- 按照角色卡的信息权限续写
- 遵循约束和文风约定

## 五、常见错误与避免

### ❌ 错误1：证据包写得太完整
**错误示例**：
```markdown
## E-001｜着陆舱环境快照
- 内容：发现了外星生命，但被指挥官隐瞒了
```

**正确示例**：
```markdown
## E-001｜着陆舱环境快照（原始数据缺段）
- 摘述：出现未知挥发物峰值（峰形异常对称）
- 明显缺口：09:13~09:15 数据段为空
- 可争论点：缺段是系统故障，还是有人提前"知道哪些数据不能留下"
```

### ❌ 错误2：立场包写成观点列表
**错误示例**：
```markdown
## S-01｜舰队指挥链
- 观点：应该保守，不要冒险
```

**正确示例**：
```markdown
## S-01｜舰队指挥链
- 解释权来源：任务章程、风险授权
- 核心利益：任务连续性、避免失控事件
- 核心恐惧：确认"智识生命"将导致撤离/隔离
- 代价结构：违令=撤职/隔离
```

### ❌ 错误3：角色卡写得太全知
**错误示例**：
```markdown
## C-01｜Sera
- 知道：所有真相
- 目标：揭露真相
```

**正确示例**：
```markdown
## C-01｜Sera
- 可接触信息：E-001~E-005（但多为摘要版）
- 不可接触信息：E-006正文
- 认知盲区：习惯把命令当作"上层一定掌握真相"
- 关键矛盾：她听到了"呼吸"，但无法证明自己没被"引导听见"
```

## 六、质量检查清单

在提交前，请检查：

- [ ] `10_evidence_pack.md` 至少包含5条证据卡
- [ ] 每条证据卡都有"明显缺口"和"可争论点"
- [ ] `20_stance_pack.md` 至少包含3个不同立场
- [ ] 每个立场都有"代价结构"
- [ ] `30_cast.md` 中每个角色都有"认知盲区"
- [ ] 角色能看到的信息和不能看到的信息明确区分
- [ ] `50_constraints.md` 明确写了硬约束
- [ ] 没有在证据包中写"结论"或"真相"
